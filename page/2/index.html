Train the linear classifier version 1

Error of a neuron

  • The error of a neuron depends on the weighting of the input, higher weighting contributes more error than the lower one

Error of the hidden layer

  • since the traning data only tells the ans of the final layer, we can only calculate the error of the final layer directly

  • but from the above, we can see the error of the hiddden layer are contributes to the nodes in final layer

  • so we can calculate the error of the hidden layer inplicily implicitly

Derive the error of hidden layer

  • The error of hidden layer node E(hidden,1), is the sum of error of W(1,1) and W(1,2)
  • so we should find out W(1,1) and W(1,2)
  • for the error of W(1,1), it only contributes a part of the error of E(output,1)
  • ∴ the error of W(1,1) = E(output,1) *(W(1,1)/(W(1,1)+ W(2,1)))
  • same for the error of W(1,2), it only contributes a part of the error of E(output,2)
  • ∴ the error of W(1,2) = E(output,2) *(W(1,2)/(W(1,2)+ W(2,2)))

Update the Weighting of a neuron

  • since we want to have accurate output of the neural network
  • which means we want to minimize the error of the output
  • so we can update the weight to minimize the error
  • There are two method to update the weight
  1. by use algebra to calculate the exact value of each weighting in the network
  • but this method is nearly impossible to done
  • since the neural network has many combinations of weights and bias in the network
  1. Gradient descent
  • try to apply random inital value to the error function
  • find the gradient of the function
  • update the Weight by a small amount (learning rate)
  • repeate the procee until it reach the lowerst point(the Weight do not change anymore)

Two mainstream to save file

serialization (save the serialized obj)

save text file (save the obj as human readable string)

Serialization

  1. create a FileOutputStream obj
  2. create a ObjOutputStream obj
  3. write obj using ObjOutputStream obj.writeObject()
  4. ObjOutputStream obj.close()

important concept

  • when serializing an obj, all the obj it refers to from instance variable are also serialized.
  • serializable interface is a marker/tag interface, there are no any method need to be implenmented
  • the interface is just used to tell the jvm the class is serializable
  • serialization is a all or nothing process
  • if any obj inside the serlizable class cannot be serialize, then the obj is not serializable
  • non serializable obj can use the key word transient to tell the jvm not serialize this obj
  • jvm will initial a new obj by the constructor for the transient obj

Deserialization

  1. create a FileInputStream obj
  2. create a ObjInputStream obj
  3. write obj using ObjOutputStream obj.readObject()
  4. casting the obj to the right obj type
  5. ObjOutputStream obj.close()

Thread in Java

Basic concept

  • Threads have indepent stack and programme counter
  • Threads share same heap, and method area
  • A java programme has at least 3 threads including main, garbage collection, and exception handle.
  • There are 2 types of threads
  • Daemon thread eg: garbage collection, and exception handle thread
  • User thread eg: main
  • Daemon will automatically stop if user thread stops

Create Thread

Method 1:

  1. inherit the Thread class
  2. override run method
  3. create obj
  4. obj.start() //not run(), run() will not create a new thread
  • Start() ->causes JVM will automatically call run method
  • one obj can only create one thread only
  • multiple objs is required to create mutilple threads

Method 2:

  1. decalre Mythread class and implement runnable interface
  2. implemrnt run method
  3. decalre Mythread obj
  4. decalre thread obj and pass Mythread obj as parameter
  5. threadObj.start()

    mutiple thread:

  • when using the same MyThreadObj to declre mutiple thread, those threads are sharing one obj with the same data (attributes)

  • tell cpu to halt the current and give the execution chance to other threads
