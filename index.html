Constructor and Garbage collector

heap and stack

  • objects live in heap (where the garbage collector work)
  • method invocations and local variables live in stack

Instance Variables (declared inside the class)

  • live in heap

Local Variables (declared inside the method)

  • live in stack
  • live cycle until the method finish execution

see below exp.
  • if a instantce variable contain a ref of another obj
  • only the ref size will be maintain in the heap, if the ref do not relate the any concrete obj
  • see below exp.

Constructor

  • name of the constructoe is same as the class name
  • constructor should not return anything
  • a class can have mutiple constructor by having diff. parameter
  • complier will not create a constructor when no any constructor is created
  • if there is a constructor, complier will not create constructor for you
  • constructor can be public, private or non assigned
  • When newing a new obj, the constrcutor will call its parent’s construct until the object class constructor.

This

  • Use this() to call a constructor from another overloaded constructor in the same class.
  • The call to this() can be used only in a constructor, and must be the first statement in a constructor.
  • A constructor can have a call to super() OR this(), but never both!

kill obj

  • when no ref. relate to an obj, it will be collect by the garbage collector
  • when memory is insufficient, GC will collect all collectable obj at once

3 ways to kill obj

  • when the obj is create inside a method, obj is killed when the method ends
  • the last ref. of an obj is assigned to another obj
  • set the last ref. of an obj to null

More about polymorphism

Be careful to the diff between Abstract class, Abstract method, interface.

Abstract class

  • Some classes are not suitable to create a instance eg. Animal, because we dont know what should a animal obj look like.
  • Abstract class help to prevent it being created as a instance obj.
  • The first concrete class must implement all the abstract methods in the class.
  • Abstract class have abstract method and non abstract method
  • see the exp below

Abstract methood

  • class contain any abstract method should declare as abstract class
  • ie. Non Abstract class CANNOT have Abstract method

Object class

  • all class extends object class
  • exp. method in obj class

    .equals() and ==

  • both compare if two refferece relate to the same obj
  • .equals() will usually be override to compare actual meaning of equality base on the obj.

interface

  • all methods are abstract
  • think interface as a contract, any class implement the interface are granteed to implenment the abstract methods
  • class can implement multiple interface
  • but can only extends from one class

Inheritance

  • child class will automatically inheritance the attributes and methods from it parents - private methods cannot be inheritated

  • cannot lower the method’s access modifer

Overload(same method name with diff parameter)

_ operator
