Head First Java Chapter 11 Exception

Posted on 2019-12-30 | In Book , Head First Java

Basic concept

  • Exception is a class

  • all exception are extend from Exception class

  • when a method declare to throws exception, we need to handle it

  • If two Exception are inherit frome the same super exception class and have the same handling method, we can catch their exception class in one catch block

  • When throwing an exception, the call of the method need to handle it, by throwing it again to the upper method(until the main) or catch it.

  • if the upper method also want to throws the exception, it should decalre the throw statement in its decoration

  • Compiler will forece programmer to handle all kinds of exceptions except the RuntimeException

  • RuntimeException is usually cause by code logic which cannot predict or prevent

Two ways to handle exception

try catch finally block

  • if return statement inside try or catch block, finally block will still be execte and then return

throws exception

  • to throw an exception simply decalre it in method decoration

Catching exception

Catching mutilple exception

  • we should catch the child exception first

Concepts

  1. catch and finally cannot be use without try
  2. no code between try and catch
  3. try must follow catch or finally
  • 4. try finally block must throw exception

Thread in Java

Posted on 2019-12-30 | In JavaSE

Basic concept

  • Threads have indepent stack and programme counter
  • Threads share same heap, and method area
  • A java programme has at least 3 threads including main, garbage collection, and exception handle.
  • There are 2 types of threads
  • Daemon thread eg: garbage collection, and exception handle thread
  • User thread eg: main
  • Daemon will automatically stop if user thread stops

Create Thread

Method 1:

  1. inherit the Thread class
  2. override run method
  3. create obj
  4. obj.start() //not run(), run() will not create a new thread
  • Start() ->causes JVM will automatically call run method
  • one obj can only create one thread only
  • multiple objs is required to create mutilple threads

Method 2:

  1. decalre Mythread class and implement runnable interface
  2. implemrnt run method
  3. decalre Mythread obj
  4. decalre thread obj and pass Mythread obj as parameter
  5. threadObj.start()

    mutiple thread:

  • when using the same MyThreadObj to declre mutiple thread, those threads are sharing one obj with the same data (attributes)

Method in thread class

Thread.currentThread()

  • get the current thread

yield()

  • tell cpu to halt the current and give the execution chance to other threads
  • the scheduling is depend on operating system not the thread

join()

  • wait until other thread to complete
  • currentThread will be blocked until other thread has completed

sleep(long millisecond)

isAlive()

Thread priority

final static int variable

  • MIN_PRIORITY = 1
  • NORM_PRIORITY = 5
  • MAX_PRIORITY = 10

getPriority()

setPriority() //set before the thread start

Thread.status

  • NEW -> not yet started
  • RUNNABLE
  • WAITING
  • TIMED_WAITING
  • TERMINATED
