ActionListener and ActionPerformed

Mechanism

  1. When user click on button, the JButton obj will automatically create a Event obj

  2. If any class want to notify by this Event obj, it should implenment the ActionListner interface and register the ActionListner in the widget

  3. ActionListener interface force will you implement the actionPerformed() method in your class

  4. When user click on button, the JButton obj will automatically create a Event obj and pass it to the class who is listening and call the actionPerformed() method automatically

Implementation Workflow

  1. implenment the ActionListener interface
  2. implenment the actionPerformed() method
  3. register the actionListener in the widget

Inner Class

  • Inner class can use all method and attributes in outter class even if they are private

  • Inner class obj is binded with the outter class obj, the instance inner obj can only access the method and attributes of the outter instance class obj who is binding with it

example of inner class to implenment the listener of two button

Create a own widget with that can paint

  1. implement the child class extends JPanel
  2. override the paintComponent() method
  • When repaint is call or when it is necessary,
  • the paintComponent menthod will call automatically by the operating system
  • the Graphics g obj is actually a Graphics2D obj
  • more method can be use by casting to Graphics2D obj
