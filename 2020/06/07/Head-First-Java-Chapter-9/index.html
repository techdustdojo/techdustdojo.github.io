Constructor and Garbage collector

heap and stack

  • objects live in heap (where the garbage collector work)
  • method invocations and local variables live in stack

Instance Variables (declared inside the class)

  • live in heap

Local Variables (declared inside the method)

  • live in stack
  • live cycle until the method finish execution

see below exp.
  • if a instantce variable contain a ref of another obj
  • only the ref size will be maintain in the heap, if the ref do not relate the any concrete obj
  • see below exp.

Constructor

  • name of the constructoe is same as the class name
  • constructor should not return anything
  • a class can have mutiple constructor by having diff. parameter
  • complier will not create a constructor when no any constructor is created
  • if there is a constructor, complier will not create constructor for you
  • constructor can be public, private or non assigned
  • When newing a new obj, the constrcutor will call its parentâ€™s construct until the object class constructor.

This

  • Use this() to call a constructor from another overloaded constructor in the same class.
  • The call to this() can be used only in a constructor, and must be the first statement in a constructor.
  • A constructor can have a call to super() OR this(), but never both!

kill obj

  • when no ref. relate to an obj, it will be collect by the garbage collector
  • when memory is insufficient, GC will collect all collectable obj at once

3 ways to kill obj

  • when the obj is create inside a method, obj is killed when the method ends
  • the last ref. of an obj is assigned to another obj
  • set the last ref. of an obj to null
